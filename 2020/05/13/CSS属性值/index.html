CSS属性值

2020.05.13 kaolalala  热度 ℃

基础环境：vue

今天在测试一个图片库功能的时候发现了一个CSS的坑。

原本我的思路是这样的，上传图片后后台返回给我一个图片的地址，我在图片库中显示用循环展示得到的图片列表。每个图片有自己的一张卡片，用css的背景图片属性把图片展示出来。

但是当我上传一张名为0 (1).jpg的时候，发现背景图片是空白的，打开控制台也发现style属性不存在了，别的卡片是有的。

这时我把图片地址在新窗口打开，发现是可以显示出来的，排除了地址错误的问题。

所以可能就是属性值的错了，我改了一下图片名字，把空格去掉，发现是可以的。

所以得出的结论是CSS属性值里最好不要带空格。我猜如果格式不正确在插值绑定时会直接被vue忽略。连style属性都没有了。

为了解决这个问题，我就换了个图片显示方法，在卡片中插入一张沾满盒子的图片<img>，src属性就是图片地址。这种情况下图片名称中有空格也不怕，我想应该是会自动把空格等特殊字符解析成编码。因为在新窗口中打开图片地址就是这样的。这样子做之前显示不出来的图片就都可以展示啦。
