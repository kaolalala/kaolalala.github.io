浏览器（2）——TCP、IP、HTTP

HTTP、TCP、IP分别是对应网络模型中的应用层、传输层、网络层的协议。传输层为应用层提供服务、网络层为应用层提供服务。

当一个web请求发出的时候，先是封装成HTTP报文，到传输层封装成TCP报文，再到网络层封装成IP数据报发送到目的地。按照对应的顺序一层一层网上解封，最后送到目的地。

其中IP协议使数据报从源IP地址到目的IP地址，每个主机都有一个自己对应的独一无二的主机号。TCP协议使数据从源IP地址上的某一个端口对应的应用程序发送到目的IP地址的某一个端口号对应的应用程序。所以IP实现的是主机之间的通信，TCP实现的是应用程序之间的通信。HTTP则是建立在TCP协议基础上的一种web之间通信协议。

TCP三次握手与四次挥手

传输层除了TCP还有另一种不可靠协议UDP。它的特点是发送数据之前不需要先建立连接，也无需确认数据是否送达目的地，相比TCP协议的优点是快，因为无需花费时间建立连接，这种比较适合像直播这类的强调实时的通信。

而TCP在发送数据之前需要先建立连接，连接的过程为著名的三次握手。发送数据时也有确认数据是否正确送达的机制。

建立连接之后HTTP才在这个基础上在客户端和服务端之间传送数据。

当通信结束想要断开连接时，需要进行四次挥手确认断开连接。

因为TCP协议有以上建立连接和断开连接的过程，所以TCP相对于UDP来说发送相同的数据花费的时间会较久，但也是比较可靠的传输。

HTTP通信

在TCP已经建立好连接的基础上，利用HTTP进行客户端和服务端之间的通信。

在准备发送请求的时候有以下六个步骤：

  1. 构建请求

  2. 查找缓存

    如果本地缓存有想要查询的信息时，就不会发送请求而是直接拿本地的数据

  3. DNS查询

    要向服务端发送请求首先得知道它的IP地址，IP地址一般由一串数字组成会比较难记，所以更多的是将一串字符地址对应一个IP地址，比如www.baidu.com对应的是183.232.231.172。这其间的映射关系可以用过DNS协议查询得到，如果浏览器在之前已经查询过域名对应的IP地址，可能会缓存在本地，那么就不需要再查询一次，直接从本地获取就可以了。

  4. 等待TCP连接

    在Chrome浏览器中，一个域名下同时最多可以建立6个TCP连接。如果当前已经超过六个那就需要等待，等到连接数不满6个才可以开始。

状态码

状态码在服务器发送响应的时候能告诉客户端处理请求的结果。类型很多，常用的200代表请求成功，404表示找不到请求的资源，需要的时候查查就知道状态码代表的含义了。
